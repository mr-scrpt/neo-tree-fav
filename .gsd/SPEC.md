# SPEC.md — Project Specification

> **Status**: `FINALIZED`

## Vision

Кастомный source для neo-tree.nvim, который позволяет создавать виртуальную структуру «Избранное» — персональную коллекцию файлов и папок из проекта. Плагин формирует данные, совместимые с внутренним форматом neo-tree, и передаёт их стандартному рендереру, переиспользуя встроенные компоненты (иконки, имена, диагностики) и команды (поиск, фильтрация, раскрытие).

## Goals

1. **Новый source `favorites`** — таб в Neo-tree, доступный как через winbar, так и через отдельный шорткат `<leader>F` (плавающее окно).
2. **Toggle-механизм** — добавление/удаление элементов в избранное клавишей `F` из источника `filesystem`.
3. **Плоский верхний уровень с коллизиями** — файлы и папки на верхнем уровне; одноимённые элементы получают хинты путей в скобках (например, `domain [src/core]`).
4. **Рекурсивное раскрытие** — избранная папка показывает своё реальное содержимое из файловой системы.
5. **Персистентность** — данные сохраняются в JSON per-project в `~/.config/nvim/favorite-projects/`.
6. **Совместимость с neo-tree** — фильтрация, fuzzy_finder и раскрытие работают «из коробки» через стандартные механизмы neo-tree.

## Non-Goals (Out of Scope)

- Собственный UI для отрисовки дерева (используем рендерер neo-tree).
- Drag-and-drop / переупорядочивание элементов в избранном.
- Избранное по удалённым (remote) путям.
- Кросс-проектный список избранного (каждый проект — отдельный файл).

## Users

Разработчики, использующие Neovim + Neo-tree, которые хотят быстрый доступ к ключевым файлам и папкам проекта без навигации по глубокой иерархии.

## Constraints

- **Язык**: Lua 5.1 (LuaJIT в Neovim).
- **Зависимость**: neo-tree.nvim v3.x.
- **Хранение данных**: `~/.config/nvim/favorite-projects/{project_name}_{cwd_hash}.json`.
- **Логирование**: `~/.config/nvim/neo-tree-favorites.log` (очистка при каждом старте).
- **Разработка**: `/home/mr/Hellkitchen/solution/nvim/neo-tree-fav` — загружается через lazy.nvim как локальный плагин, в финале — через GitHub.
- **Архитектура**: Clean Architecture — разделение на слои данных, хранения и интеграции с neo-tree.
- **Стратегия разработки**: Mocks First — сначала добиться корректной визуализации, затем реальная логика.

## Success Criteria

- [ ] `<leader>F` открывает плавающее окно с деревом Favorites.
- [ ] Favorites доступен как таб в winbar рядом с filesystem, buffers, git_status.
- [ ] Клавиша `F` в filesystem добавляет/удаляет элемент из избранного (toggle).
- [ ] Коллизии имён разрешаются хинтами путей: `domain [src/core]` vs `domain [src/modules/users]`.
- [ ] Избранная папка при раскрытии показывает реальное содержимое FS.
- [ ] Данные сохраняются и восстанавливаются между сессиями (per-project JSON).
- [ ] Встроенный поиск/фильтрация neo-tree работает корректно с виртуальными узлами.
